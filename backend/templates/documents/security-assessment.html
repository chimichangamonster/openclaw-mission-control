6. Testing Methodology

This assessment followed a structured methodology aligned with industry standards (PTES, OWASP, NIST SP 800-115).

  1. Reconnaissance & OSINT — Domain enumeration, subdomain discovery, DNS record analysis, email harvesting, doppelganger domain scanning, public data exposure checks.
  2. Host Discovery — Network scanning (ping sweeps, TCP/UDP port scans) to identify live hosts, open ports, and running services within the defined scope.
  3. Enumeration — Service fingerprinting, version detection, protocol analysis, SSL/TLS configuration review, directory enumeration, and banner grabbing.
  4. Vulnerability Analysis — Identification and validation of vulnerabilities through automated scanning and manual testing. CVSS 3.1 scoring applied to each confirmed finding.
  5. Exploitation — Controlled exploitation of confirmed vulnerabilities to demonstrate real-world impact. Only safe exploits executed per Rules of Engagement.
  6. Post-Exploitation — {% if post_exploitation_allowed %}Privilege escalation, lateral movement, and data access assessment where initial exploitation succeeded.{% else %}Post-exploitation was limited or excluded per the Rules of Engagement for this engagement.{% endif %}
  7. Reporting — Findings documented with evidence, CVSS scoring, MITRE ATT&CK mapping, and prioritized remediation guidance.
